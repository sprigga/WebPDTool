Code review 是確保程式碼品質的重要環節，以下是幾個關鍵面向：
1.功能正確性是首要考量。檢查程式碼是否真正解決了問題，邏輯是否正確，有沒有遺漏的邊界條件或錯誤處理。特別要注意 null/undefined 檢查、陣列越界、除零等常見問題。
2.可讀性與維護性同樣重要。程式碼應該清晰易懂，變數和函數命名要有意義，複雜邏輯需要適當註解說明。問問自己：六個月後其他人（或你自己）能快速理解這段程式碼嗎？
3.在程式架構方面，要檢查是否符合專案的設計模式和架構原則，函數是否職責單一，模組間的耦合度是否適當，有沒有重複的程式碼可以抽取成共用函數。
4.安全性不容忽視。特別注意 SQL injection、XSS、CSRF 等常見漏洞，檢查輸入驗證、權限控制、敏感資料處理是否妥當。以你的背景來說，OT/IT 整合場景中的安全問題更需要特別留意。
5.效能考量包括是否有不必要的迴圈嵌套、資料庫查詢是否優化（N+1 問題）、記憶體使用是否合理等。在處理大量資料或高頻操作時這點特別重要。
6.測試覆蓋率也要關注。新增的功能是否有對應的單元測試？測試案例是否涵蓋了正常情況和異常情況？
